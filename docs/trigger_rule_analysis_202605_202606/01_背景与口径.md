# 01 · 背景与口径

## 1.1 业务背景

2026年6月，为获取足够的统计分析样本，项目对**全部方案**启用了**自动触发规则**。本报告基于这一个月的触发与结算数据，结合来源项目（AITradingSimulator）5–6月的预测历史回测，判断：

- 哪些方案适合哪条规则、哪些不适合；
- 6/22 规则1阈值调整的影响；
- 结合5月数据，方案表现是否稳定。

## 1.2 两条自动触发规则

### 规则1「低命中自动跟单」（回补逻辑）
- **当前阈值（6/22起）**：近100期 `combo<16%` 或 `big_small<38%` 或 `odd_even<38%`，任一命中即触发（`condition_mode=any`）
- **旧阈值（6/7–6/21）**：近100期 `combo<20%` 或 `big_small<40%` 或 `odd_even<40%`
- `min_sample_count=50`，`scope=all_subscriptions`（sub 2,5,6,7）
- 逻辑假设：方案进入"冷区"后，命中率会**回补**反弹，此时开周期跟单。
- action：`restart_cycle` + `play_filter_action=matched_metric` + `fixed_metric=big_small` + `skip_multiple_metrics_matched=true`
- 风控：profit_target=20，loss_limit=20

### 规则2「高命中自动跟单」（动量/赔率逻辑）
- **阈值**：近100期 `combo≥35%` 或 `odd_even≥60%` 或 `big_small≥60%`，任一命中即触发
- `scope=all_subscriptions`（sub 2,4,5,6,7）
- 逻辑假设：方案进入"热区"后，命中率会**延续**（动量），开周期跟单。
- action：`restart_cycle` + `play_filter_action=matched_metric` + `fixed_metric=big_small` + `skip_multiple_metrics_matched=false`
- 风控：profit_target=20，loss_limit=20

## 1.4 赔率与盈亏平衡线（关键）

结算规则为 `pc28_high_regular`（PC28 高赔常规，profit_rule=`pc28_high`，odds_profile=`regular`）：

| 指标 | 赔率 | 净赔 | 盈亏平衡胜率 |
|---|---|---|---|
| big_small | 2.846 | 1.846 | **35.1%** |
| odd_even | 2.846 | 1.846 | **35.1%** |
| combo（小单/大双组） | 6.78 | 5.78 | 14.7% |
| combo（大单/小双组） | 6.33 | 5.33 | 15.8% |
| combo 综合（取约6.5） | ~6.5 | ~5.5 | **15.4%** |

**结论：bs/oe 盈亏线 35.1%，combo 盈亏线约 15.4%。** 这是后续所有 ROI 评价的基准。

> ⚠️ 若改用 `pc28_netdisk_regular`（bs/oe 赔1.98→盈亏线50.3%；combo 赔3.6→盈亏线27.8%），盈亏线大幅抬高，绝大多数组合将转亏。本报告结论**仅适用高赔规则**。

## 1.5 数据口径

- **本仓真实结算**：`subscription_progression_events` 表，`auto_trigger_rule_id` 非空的已结算事件（hit/miss/refund），按 `auto_trigger_stat_date` 分月、按 `result_context_json->metric` 分指标。
- **来源仓回测**：`pc28_predictor.db` 的 `predictions` 表（5–6月约14万条已结算），按 `predictor_id × hit_{metric}` 计算滑动100期命中率与触发后未来40期命中率。
- ROI = 净盈亏 / 总下注；胜率 = hit / (hit+miss)（refund 不计入分母，但计入下注与盈亏）。
