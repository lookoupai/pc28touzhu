# 加拿大28 三种玩法规则与赔率速查

> 存档时间：2026-07-01
> 用途：以后不用再翻聊天记录找玩法规则与赔率。本仓实际使用 **正常玩法（赔率区，非abc玩法）**。
>
> 三种玩法共享同一套"开奖判定"逻辑，差异只在**赔率**和**命中回本规则**。

---

## 0. 一句话区分三种玩法

| 玩法 | 结算规则ID | 大小单双赔率 | 组合赔率 | 命中回本规则 |
|---|---|---|---|---|
| **高倍** | `pc28_high_regular` | 2.846 | 6.78 / 6.33 | 命中遇13/14+对子+顺子+豹子 → 回本 |
| **网盘** | `pc28_netdisk_regular` | 1.998 | 3.71 / 4.32 | **无任何回本**（13/14也正常赔） |
| **2.0** | （代码未内置，文本规则）| 2（13/14返1.96） | 4.73 / 4.31 | 仅13/14命中回本；对子/顺子/豹子正常赔 |

> **本仓当前6个订阅全部使用 `pc28_high_regular`（高倍），已验证为最优玩法（见 `06_止盈效果口径与玩法对比.md`）。**

## 1. 共同的开奖判定（三种玩法通用）

PC28 = 加拿大28，三个数字（A、B、C，各 0–9）相加得总和（0–27）。

| 玩法项 | 判定 |
|---|---|
| 🟧大 / 🔸小 | 总和 0–13 为小，14–27 为大 |
| ✋单 / 🤲双 | 总和奇数为单，偶数为双 |
| 组合 | 大小+单双组合：小单 / 小双 / 大单 / 大双 |
| 极值 | 极小 0–5，极大 22–27 |
| 对子 | 三个数字有两个相同 |
| 顺子 | 三个数字相邻（见下方各玩法差异） |
| 豹子 | 三个数字相同 |
| 数字 | 三个数字总和 0–27 |

### 组合概率（实测2万期，用于期望计算）
| 组合 | 概率 |
|---|---|
| 小单 | 26.8% |
| 大双 | 27.0% |
| 大单 | 23.0% |
| 小双 | 23.2% |

### 特殊牌型频率（实测2万期）
| 牌型 | 频率 |
|---|---|
| 总和=13或14 | 7.4%（13和14各约3.7%）|
| 对子 | 27.1% |
| 顺子 | 4.9%（高倍含0,1,9；网盘/2.0不含0,1,9，约3.5%）|
| 豹子 | 1.1% |

## 5. 三玩法核心对比（一眼选型）

### 正常玩法区赔率对比

| 玩法 | 大小/单双 | 小单/大双 | 大单/小双 | 顺子判定 | 命中回本 |
|---|---|---|---|---|---|
| **高倍** | **2.846** | **6.78** | **6.33** | 含0,1,9 | 13/14+对子+顺子+豹子 |
| 网盘 | 1.998 | 3.71 | 4.32 | 不含0,1,9 | **无** |
| 2.0 | 2（13/14返1.96）| 4.73 | 4.31 | 不含0,1,9 | 仅13/14 |

### 等效净赔对比（去掉回本后的真实赔付，决定止盈效果）

| 指标 | 高倍 | 网盘 | 2.0 | 最优 |
|---|---|---|---|---|
| 大小/单双 | **1.43** | 0.998 | 0.985 | **高倍** |
| 组合 | **3.46** | 3.32 | 3.31 | **高倍** |

### 止盈效果对比（触发后开周期+20/-20/200期，实测均值）

| 玩法 | 平均止盈率 | 平均ROI |
|---|---|---|
| **高倍** | **34%** | **+0.96%** |
| 网盘 | 22% | +0.29% |
| 2.0 | 23% | +0.15% |

**结论：高倍玩法整体最优，本仓当前配置正确。**
- 高倍的高赔率足以覆盖回本损失，等效净赔仍最高
- 网盘/2.0 赔率较低，无回本补偿不了，期望近零或略负
- 详细回测见 `06_止盈效果口径与玩法对比.md`

## 6. 代码实现位置

| 配置 | 位置 |
|---|---|
| 结算规则定义 | `src/pc28touzhu/domain/settlement_rules.py` `SETTLEMENT_RULE_CATALOG` |
| 赔率定义 | `src/pc28touzhu/domain/pc28_profit_rules.py` `PC28_PROFIT_RULES` |
| 回本判定 | `settlement_rules.py` `_resolve_refund_aware_result()` |
| 顺子判定 | `settlement_rules.py` `_is_pc28_straight()`（严格相邻，不含0,1,9）|

### 代码中已内置的结算规则
- `pc28_netdisk_regular` / `pc28_netdisk_abc`（网盘）
- `pc28_high_regular` / `pc28_high_abc`（高倍）

> 2.0 玩法代码未内置，若需使用要新增 `SETTLEMENT_RULE_CATALOG` 条目（`profit_rule_id` 新增 `pc28_2`、赔率档 + `refund_policy` 仅13/14）。

### 顺子判定注意
代码 `_is_pc28_straight()` 是**严格相邻**（0,1,2…7,8,9），**不含0,1,9**。
- 与网盘/2.0规则一致
- 与高倍规则文本"890/190算顺子"（含0,1,9）有出入——若严格按高倍文本，代码需调整 `_is_pc28_straight` 在高倍场景包含 {0,1,9}。当前代码对高倍也是不含0,1,9，实测影响很小（0,1,9组合频率约1.4%）。
